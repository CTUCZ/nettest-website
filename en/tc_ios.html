Privacy Policy and Terms of Use for iOS

The mobile application of CTU-NetTest can be used only after explicit consent to CTU’s Privacy Policy and Terms of Use for the CTU-NetTest.

1 General

The CTU-NetTest mobile application of the Czech Telecommunication Office (hereinafter “CTU”) informs users about the current QoS (including upload, download and ping) of their Internet connection. CTU-NetTest provides also further possibilities to users with wider technical interests: CTU-NetTest offers also the option to repeat test automatically (loop mode). Furthermore, it is possible to use the loop mode only for signal for which it is possible to repeatedly measure only signal parameters, without measuring the speed and QoS.

CTU’s legal authorisation to offer the CTU-NetTest, including the possible processing of personal data within the CTU-NetTest, is enshrined in Article 4 (4) and Article 5 (1) Regulation (EU) 2015/2120 (hereinafter “EU Regulation”). To fulfill this duty and ensure users with the greatest possible transparency and information – while respecting their privacy – the CTU-NetTest is based on the open data and open source principles.

2 Which data are processed?

2.1 CTU-NetTest and selection of loop mode and certified measurement

The following data are processed as part of the CTU-NetTest:

  • actually achieved speed of data connection in both directions (download/upload)*,
  • roundtrip delay of data connection (ping)*,
  • test parameters (test method, test status, test server, test configuration)*,
  • randomly generated client ID (Client UUID),
  • evidence of acceptance of the Privacy Policy and Terms of Use,
  • public client ID*,
  • client sync code and group,
  • randomly generated ID of the measurement series (Loop UUID),
  • test start time,
  • automatically determined client location (if known) / time zone*,
  • exact client location (determined by the optional manually entered address of the measurement),
  • anonymised (dissected) location (determined by the optional manually entered address of the measurement)*,
  • meta information for the client location (e.g. location accuracy, method of determining the location, distance)*,
  • device type/model, operating system/software version of browser*,
  • NAT Yes/No (direct Internet access or private address used)*,
  • IP version*,
  • IP address,
  • AS name and number (autonomous system) or anonymised IP address*,
  • WLAN network ID (SSID) and numeric WLAN ID (BSSID),
  • Wireless network, e.g., home network, mobile network, connection type (GSM, UMTS, LTE, 5G etc.), mobile network name, frequency used, base station‘s identification*.

The data marked with asterisk * are published on the CTU-NetTest website and are publicly available as open data for information, use, dissemination and repetitive use.

Note: After providing consent with „Privacy Policy and Terms of Use“, to authorise the use on devices with more SIM cards, a message „Allow CTU-NetTest to initiate and manage calls“ (dependently on the device’s operation system). Thus, the identification of a SIM card currently used for mobile data is allowed within the context of CTU-NetTest use. The CTU-NetTest does not use any other functions authorised this way.

2.2 Loop mode

The loop mode allows to automatically carry out repeated measurement with the CTU-NetTest tool by personalised user setting.

The first test is started after switching on the loop mode. Next tests are started after the waiting time (the default time is 15 minutes) passes or after passing the given distance (the default distance is 250 metres). Tests will be running until a requested number of tests is reached (the default number of tests is 10). The level of signal is measured continuously, even in between of two tests.

2.3 Loop mode only for signal

The loop mode only for signal allows to automatically carry out repeated measurement of signal parameters also without measuring the speed or QoS. The user can stop the loop mode anytime or it is automatically switched off after 48 hours.

The data processed is the same as listed in point 2.1 with exception of roundtrip delay (ping), signal level, testing communication on selected network ports, content modification, reference website transmission test, background data traffic, server DNS requirement, traceroute, VoIP test; Therefore, no personal data are processed.

3 Processing and the purposes of personal data

3.1 Processing and the purposes of personal data

As a part of the CTU-NetTest and its loop mode and loop mode only for signal, the following personal data categories are processed: (1) the Client UUID, (2) WLAN network ID (SSID) and numeric WLAN ID (BSSID), and (3) the IP address.

Purposes for processing these personal data:

  • client UUID: possibility of compiling a test result history for the user; identifying associated tests to identify improper use or malfunctions; exercising of the user´s rights according to item 5.
  • WLAN network ID (SSID) and numerical WLAN ID (BSSID): presentation in the user’s history; identifying associated tests to identify improper use or malfunctions.
  • IP address: Technical enabling of the CTU-NetTest and ensuring the full functionality, in particular display of the map and quality measurements; in order to be able to assign the tests to individual service providers (via routing information – autonomous system number or host names); detection of improper use or malfunctions..

4 Duration of storage

The IP address, WLAN network ID (SSID) and numeric WLAN ID (BSSID) is stored by CTU for a maximum period of six months.

5 Rights in connection with personal data

By providing the randomly generated client ID (Client UUID) - which can be found under the Options bookmark – the user has amongst others the following rights concerning her or his personal data:

  • access to personal data,
  • erasure, applied through the right to withdraw their acceptance to the processing of their personal data at any time,
  • submit a complaint with the Office for Personal Data Protection (https://www.uoou.cz/),
  • rectification,
  • restriction of the processing,
  • data portability.

6 Contact

Questions regarding the processing of personal data and data protection in connection with the CTU-NetTest can be sent to: osobniudaje@ctu.cz.

The CTU-NetTest of the Czech Telecommunication Office (hereinafter “CTU”) informs users about the current QoS (including upload, download and ping) of their Internet connection. Additionally, CTU-NetTest offers also the option to repeat test automatically (loop mode). Furthermore, in case a provider provides the IAS on the area of the Czech Republic, the test offers the possibility to carry out a certified measurement. With this, conclusive findings about the quality of the IAS can be made. These findings enable the user to activate remedies available in compliance with national legislation in case of any significant and continuous or regularly recurring discrepancy of actual service performance in terms of upload and download speed, from the performance the IAS provider stated in the contract.
